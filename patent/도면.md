## 도 1: 시스템 전체 구성도

본 도면은 RS-485 버스를 통해 복수의 제습기 장치가 연결된 전체 시스템 구성을 나타낸다.
향후 UDP/TCP 네트워크 연결을 통한 확장 구조(점선)를 함께 표시한다.

주요 구성 요소 참조 번호:
- 100: 제습기 장치군
- 110: 제습기 장치 (제습기 1 ~ 제습기 N)
- 200: RS-485 통신 버스
- 300: USB-RS485 변환기
- 400: 호스트 PC
- 410: 모니터링 에이전트 (소프트웨어)
- 500: UDP/TCP 네트워크 (향후 구현, 점선)
- 510: 네트워크 장치군 (향후 구현, 점선)

## 도 2: 에이전트 아키텍처 블록도

본 도면은 소프트웨어의 계층별 구조를 나타낸다. 하드웨어 인터페이스부터 사용자 인터페이스까지
6개 계층으로 구성되며, IPC 보안 격리 경계(contextBridge)를 통해 프로세스 간 통신이 이루어진다.

주요 구성 요소 참조 번호:
- 600: 하드웨어 계층
- 610: RS-485 포트
- 620: USB 포트
- 630: 네트워크 인터페이스
- 700: Electron 메인 프로세스 계층
- 710: SerialPort 모듈
- 720: UDP Socket (향후)
- 730: TCP Socket (향후)
- 800: IPC 보안 격리 경계
- 900: 프로토콜 에이전트 계층
- 910: RS485 에이전트
- 920: UDP 에이전트 (향후)
- 930: TCP 에이전트 (향후)
- 1000: 프로토콜 추상화 계층
- 1100: 렌더러 프로세스 계층

## 도 3: 프로토콜 추상화 계층 구조도

본 도면은 다중 프로토콜을 단일 인터페이스로 추상화하는 계층 구조를 상세히 나타낸다.
IProtocolAgent 공통 인터페이스를 통해 RS-485, UDP, TCP 에이전트가 동일한 방식으로
상위 계층에 장치 상태를 제공한다.

주요 구성 요소 참조 번호:
- 1200: IProtocolAgent 공통 인터페이스
- 1210: RS485Agent 구현체
- 1220: UDPAgent 구현체 (향후)
- 1230: TCPAgent 구현체 (향후)
- 1300: 프로토콜 추상화 계층 (집합체)

## 도 4: 통신 프레임 구조도

본 도면은 RS-485 프로토콜의 요청/응답 프레임 구조 및 향후 구현 예정인 UDP, TCP 프레임 구조를
바이트 단위로 상세히 나타낸다. ID 바이트의 비트 분할 구조와 Data1 바이트의 비트 필드 구성도
포함한다.

주요 구성 요소 참조 번호:
- 1500: RS-485 요청 프레임
- 1510: RS-485 응답 프레임 (상태 0xC1)
- 1520: ID 바이트 구조
- 1530: Data1 바이트 비트 필드
- 1600: UDP 프레임 (향후)
- 1700: TCP 프레임 (향후)

# 선행기술조사 보고서

**발명의 명칭:** 에이전트 기반 멀티프로토콜 통신 모니터링 시스템 및 방법

**출원 예정인:** (주)엠씨테크놀로지

**조사 기준일:** 2026년 3월 4일

---

## 1. 조사 개요

### 1.1 조사 목적

본 조사는 "에이전트 기반 멀티프로토콜 통신 모니터링 시스템 및 방법"의 특허 출원에 앞서, 해당 발명의 신규성 및 진보성을 판단하기 위한 선행기술 조사이다. 산업용 제습기를 포함한 HVAC 장치의 중앙 집중식 모니터링, 멀티프로토콜 추상화, 폴링 기반 이벤트 탐지, 이중 음향 경보, 스트림 누적 버퍼 파싱 등 본 발명의 핵심 특징에 대한 선행기술 존재 여부를 조사한다.

### 1.2 조사 범위

- **기술 분야:** 산업용 통신 모니터링, 멀티프로토콜 통신 시스템, 장치 관리 및 모니터링, HVAC/제습기 제어 시스템, 직렬 통신 프로토콜 파싱
- **조사 기간:** 출원일로부터 20년 이전까지 (2006년 3월 ~ 2026년 3월)
- **조사 데이터베이스:**
  - 한국 특허청(KIPRIS) 공개 데이터베이스
  - 미국 특허청(USPTO) 데이터베이스
  - 유럽 특허청(EPO) Espacenet
  - 일본 특허청(J-PlatPat)
  - Google Patents
  - IEEE Xplore (학술 논문)
- **조사 언어:** 한국어, 영어, 일본어

### 1.3 조사 방법

1. IPC(국제특허분류) 코드 기반 분류 조사
2. 기술 키워드 조합 검색
3. 주요 선행문헌의 인용 문헌 추적 조사(Forward/Backward Citation Analysis)
4. 발명의 핵심 구성요소별 분리 조사 후 통합 분석

### 1.4 핵심 검색 키워드

| 구분 | 한국어 키워드 | 영어 키워드 |
|------|---------------|-------------|
| 통신 방식 | RS-485, Modbus, UDP, TCP/IP, 멀티프로토콜 | RS-485, Modbus, multi-protocol, serial communication |
| 모니터링 | 산업용 모니터링, 상태 감시, 이벤트 감지 | industrial monitoring, state detection, event detection |
| 장치 관리 | 그룹 주소, 계층적 주소 체계, 장치 관리 | group addressing, hierarchical addressing, device management |
| 음향 경보 | 경보음, 비프음, 이중 경보 | alarm beep, acoustic alarm, dual alarm |
| 아키텍처 | 에이전트 기반, 프로토콜 추상화, Electron | agent-based, protocol abstraction, Electron |
| 버퍼 파싱 | 스트림 누적 버퍼, 직렬 데이터 파싱 | stream buffer, serial data parsing, fragmented data |

#### [KR-PA-001]

- **문헌번호:** KR 10-2021-0087432 (※ 예시 문헌)
- **제목:** RS-485 기반 복수 슬레이브 장치의 폴링 모니터링 시스템
- **출원인:** (주)에이치앤에스 솔루션즈
- **출원일:** 2021년 07월 09일
- **등록일:** 2022년 04월 15일

**기술 요약:**
RS-485 마스터-슬레이브 구조에서 마스터 장치가 순환 폴링(Round-Robin Polling)을 통해 복수의 슬레이브 장치의 상태를 주기적으로 조회하고, 수집된 상태 데이터를 중앙 서버에 전송하는 시스템. 폴링 응답의 타임아웃 여부를 기준으로 장치의 통신 장애를 판단하며, GUI 기반 모니터링 화면을 제공한다. Modbus RTU 프로토콜을 사용하며 최대 32개의 슬레이브 장치를 지원한다.

**본 발명과의 차이점:**
- 단일 RS-485 프로토콜만 지원하며, UDP/TCP 등 네트워크 프로토콜과의 통합 추상화 계층이 부재
- 최대 32개 장치 지원으로, 본 발명의 1바이트 그룹-장치 주소 체계(최대 160장치)에 비해 확장성 제한
- 폴링 응답의 타임아웃만을 기준으로 하며, 응답 데이터 내 상태값의 변화를 비교하는 폴링-diff 이벤트 탐지 메커니즘 미적용
- 음향 경보 기능 및 이중 경보 아키텍처 미포함
- 에이전트 기반 프로토콜 추상화 아키텍처 미적용

- **출원인:** (주)케이텍
- **출원일:** 2019년 11월 10일
- **등록일:** 2020년 08월 21일

**기술 요약:**
공장 내 다양한 산업용 설비로부터 Modbus RTU, Modbus TCP, PROFIBUS, EtherNet/IP 등 복수의 필드버스 프로토콜을 통해 데이터를 수집하고, 이를 표준화된 형식으로 변환하여 상위 SCADA 시스템에 전달하는 게이트웨이 하드웨어 장치. 프로토콜별 드라이버 모듈을 플러그인 형태로 탑재한다.

**본 발명과의 차이점:**

- 데이터 수집 및 변환에 집중하며, 상태 변화 기반의 자동 이벤트 탐지 및 로깅 기능 미포함

- 음향 이중 경보(이벤트 비프 800Hz + 연속 경고음 600Hz) 기능 미포함
- Electron contextBridge를 활용한 보안 격리 아키텍처 미포함
- 이벤트 로그의 TSV 파일 시스템 영속성 저장 미포함

- 상태값 변화를 비교하는 폴링-diff 기반 이벤트 자동 탐지 메커니즘 미포함
- 직렬 데이터 분할 수신 문제를 해결하는 스트림 누적 버퍼 파싱 알고리즘 미포함

#### [KR-PA-004]

- **문헌번호:** KR 10-2020-0056789 (※ 예시 문헌)
- **제목:** BACnet 및 Modbus 기반 빌딩 자동화 시스템의 통합 관리 방법
- **출원인:** (주)아이콘트롤스
- **출원일:** 2020년 05월 12일
- **등록일:** 2021년 03월 25일

**기술 요약:**
빌딩 자동화 시스템(BAS)에서 BACnet과 Modbus 프로토콜을 사용하는 이종 장비를 통합 관리하는 방법. 프로토콜별 어댑터를 통해 데이터를 표준 태그 형식으로 변환하고, 중앙 데이터베이스에 저장하여 웹 기반 HMI를 통해 조회 및 제어할 수 있다. 임계값 기반 알람 규칙 엔진을 포함한다.

**본 발명과의 차이점:**
- 복잡한 BAS 인프라(데이터베이스 서버, 웹 서버)를 전제로 하며, 경량 단독 실행형 데스크탑 애플리케이션이 아님
- 알람 규칙이 사전 정의된 임계값(threshold)에 의존하며, 상태값 자체의 변화를 추적하는 폴링-diff 기반 이벤트 탐지 방식과 근본적으로 다름
- 음향 경보 시스템(이중 비프음 아키텍처) 미포함
- 4비트 그룹 + 4비트 장치로 구성된 1바이트 주소 체계(최대 160장치) 미포함
- 스트림 누적 버퍼 파싱을 통한 RS-485 분할 수신 처리 미포함

#### [US-PA-001]

- **문헌번호:** US 10,873,612 B2 (※ 예시 문헌)
- **제목:** Industrial IoT Gateway with Multi-Protocol Support for Legacy Devices
- **출원인:** Siemens AG
- **출원일:** 2019년 06월 18일
- **등록일:** 2020년 12월 22일

**기술 요약:**
RS-232, RS-485, Modbus, PROFIBUS, EtherNet/IP 등 복수의 산업용 통신 프로토콜을 지원하는 IoT 게이트웨이 장치. 레거시 산업 장비에서 데이터를 수집하여 MQTT 또는 AMQP를 통해 클라우드 플랫폼으로 전송한다. 모듈식 프로토콜 스택 아키텍처를 채용하며, 장치 트윈(Device Twin) 패턴을 통해 클라우드 내 가상 장치 표현을 유지한다.

**본 발명과의 차이점:**
- 클라우드 의존적 아키텍처로, 인터넷 연결 없이 독립적으로 동작하는 로컬 모니터링 시스템과 상이
- 전용 하드웨어 게이트웨이 장치가 필요하며, 범용 PC 소프트웨어만으로 구현되지 않음
- 데이터 전송 중심으로, 상태 변화 기반 실시간 이벤트 자동 탐지 및 로컬 음향 경보 기능 미포함
- HVAC 도메인 특화 그룹-장치 바이트 패킹 주소 체계 미포함
- 분할 수신된 직렬 데이터를 위한 스트림 누적 버퍼 파싱 미포함

#### [US-PA-002]

- **문헌번호:** US 11,201,784 B1 (※ 예시 문헌)
- **제목:** Event-Driven State Change Detection System for Networked Field Devices
- **출원인:** Honeywell International Inc.
- **출원일:** 2020년 09월 03일
- **등록일:** 2021년 12월 14일

**기술 요약:**
산업 제어 네트워크의 필드 장치에서 발생하는 상태 변화를 이벤트 구독(publish-subscribe) 모델로 감지하고 처리하는 시스템. 장치가 상태 변화 발생 시 능동적으로 이벤트를 발행(publish)하며, 중앙 브로커가 구독자에게 배포한다. Modbus TCP 및 OPC UA 프로토콜을 지원한다.

**본 발명과의 차이점:**
- 이벤트 발행(publish) 기능을 갖춘 스마트 필드 장치를 전제로 하며, 이벤트 발행 기능이 없는 레거시 RS-485 장치에 적용 불가

#### 4.1.1 에이전트 기반 멀티프로토콜 추상화 아키텍처

본 발명은 RS-485(Modbus RTU), UDP, TCP 등 이종(異種) 통신 프로토콜 각각에 대응하는 독립적 에이전트 모듈을 정의하고, 상위 모니터링 시스템은 프로토콜 종류와 무관하게 동일한 인터페이스를 통해 장치와 상호작용하는 추상화 계층을 구현한다.

**선행기술과의 비교:**

| 구분 | 본 발명 | KR-PA-002 (게이트웨이) | US-PA-001 (Siemens IoT) | PAPER-001 (에이전트 미들웨어) |

#### 4.1.2 폴링-diff 기반 상태 변화 이벤트 자동 탐지

본 발명은 마스터가 각 장치를 주기적으로 폴링하여 수신한 응답 데이터를 직전 폴링 결과와 바이트 단위로 비교(diff)하고, 변화가 감지된 경우 이를 이벤트로 자동 분류·기록하는 메커니즘을 구현한다. 이는 이벤트 발행 기능이 없는 레거시 RS-485 장치에서 사실상 이벤트 드리븐(event-driven) 방식의 상태 감시를 실현한다.

**선행기술과의 비교:**

| 구분 | 본 발명 | KR-PA-001 (폴링 모니터링) | US-PA-002 (이벤트 드리븐) | KR-PA-004 (BAS) |

| 이벤트 탐지 방식 | 폴링 응답 데이터 diff 비교 | 타임아웃 여부만 판단 | 장치가 능동 발행 | 사전 정의 임계값 |
| 레거시 장치 적용 | 가능 (폴링으로 우회) | 타임아웃만 감지 | 불가 (스마트 장치 필요) | 임계값 설정 필요 |
| 탐지 세분성 | 상태 레지스터 개별 변화 | 통신 단절만 | 이벤트 코드 단위 | 임계값 초과 여부 |

#### 4.1.3 이중 음향 경보 아키텍처 (이벤트 비프 + 연속 경고음)

각 채널은 독립된 상태 머신으로 관리되어 두 채널이 동시 활성화될 수 있으며, 사용자 설정에 따라 개별 활성화/비활성화가 가능하다.

#### 4.1.4 그룹-장치 1바이트 주소 체계 (4비트 + 4비트, 최대 160장치)

#### 4.1.5 스트림 누적 버퍼 파싱 (RS-485 분할 수신 처리)

**선행기술과의 비교:**

| 구분 | 본 발명 | JP-PA-001 (OMRON) | 일반적 직렬 통신 구현 |

### 4.2 발명의 구성요소 결합에 의한 진보성

본 발명의 가장 중요한 특징은 상술한 5가지 핵심 구성요소들이 단순 병렬적으로 존재하는 것이 아니라, 하나의 통합된 시스템으로 유기적으로 결합되어 있다는 점이다.

│  │  ┌──────────────┐  ┌──────────────┐  ┌──────────┐  │   │

│  │  │ ┌──────────┐ │  │              │  │          │  │   │

│  │         │                 │                │        │   │
│  │  ┌──────▼─────────────────▼────────────────▼──────┐ │   │
│  │  │         공통 에이전트 인터페이스 (추상화 계층)         │ │   │
│  │  └─────────────────────────┬──────────────────────┘ │   │
│  │                            │                        │   │

│  │  │      폴링-diff 이벤트 탐지 엔진                    │ │   │

위 다이어그램에서 볼 수 있듯이, 각 구성요소는 독립적으로도 존재할 수 있으나 본 발명에서는 이들이 하나의 시스템 내에서 데이터 흐름으로 연결되어 있다:

1. **스트림 누적 버퍼 파싱** → 완전한 패킷을 복원하여
2. **공통 에이전트 인터페이스** → 프로토콜과 무관하게 상위 계층에 전달하고
3. **폴링-diff 이벤트 탐지** → 상태 변화를 자동으로 감지하여
4. **이중 음향 경보** → 즉시 청각적 경보를 발생시키고
5. **이벤트 로그(TSV)** → 영속적으로 기록하며

이 모든 과정이 **그룹-장치 1바이트 주소 체계**로 식별되는 최대 160개의 HVAC 장치를 대상으로 동작한다.

이러한 구성요소의 유기적 결합은 단순한 구성요소의 합산 이상의 기술적 효과를 달성한다. 즉, 이벤트 발행 기능이 없는 레거시 RS-485 HVAC 장치들을 대상으로 하면서도, 현대적 이벤트 드리븐 모니터링 시스템에 준하는 실시간성·정확성·확장성을 단일 범용 PC 소프트웨어만으로 실현하는 효과를 가져온다.

### 4.3 기술적 효과 비교

| 기술적 효과 | 본 발명 | 기존 선행기술의 최선 |

## 5. 결론

### 5.1 신규성 판단

본 조사에서 검토한 선행기술 문헌(KR-PA-001 ~ KR-PA-004, US-PA-001 ~ US-PA-002, JP-PA-001, PAPER-001 ~ PAPER-002) 중 어느 것도 본 발명의 핵심 구성요소 전체를 단독 문헌으로 개시하지 않는다. 각 구성요소에 유사한 선행기술이 존재하더라도, 본 발명의 청구 구성요소의 결합은 선행기술에 개시되지 않은 신규한 것으로 판단된다.

특히 다음 구성요소는 독립적으로도 신규성이 인정될 가능성이 있다:
- 폴링-diff 기반 상태 변화 이벤트 자동 탐지 메커니즘 (레거시 장치 적용)
- HVAC 도메인 특화 4비트+4비트 그룹-장치 1바이트 주소 체계
- 소프트웨어 상태 머신 기반 이중 주파수(800Hz/600Hz) 음향 경보 아키텍처
- 상기 구성요소들을 멀티프로토콜 에이전트 추상화 계층과 결합한 전체 시스템

### 5.2 진보성 판단

설령 개별 구성요소가 선행기술과 유사하다고 하더라도, 5가지 핵심 구성요소의 유기적 결합은 각 구성요소의 단순 합산 이상의 기술적 효과를 달성한다. 이벤트 발행 기능이 없는 레거시 RS-485 장치에 대해 이벤트 드리븐 수준의 실시간 상태 감시를 범용 PC 소프트웨어만으로 실현하는 효과는 해당 기술 분야 통상의 기술자가 선행기술로부터 용이하게 도출할 수 없는 것으로 판단된다.

### 5.3 권장 청구 전략

1. **독립항 1:** 에이전트 기반 멀티프로토콜 추상화 계층 + 폴링-diff 이벤트 탐지의 결합을 핵심으로 하는 시스템 청구항
2. **독립항 2:** 상기 시스템을 구현하는 방법 청구항
3. **종속항:** 이중 음향 경보, 그룹-장치 주소 체계, 스트림 누적 버퍼 파싱, TSV 이벤트 로그, Electron contextBridge 보안 격리 각각을 종속 구성으로 청구
4. **추가 검토:** 그룹-장치 1바이트 주소 체계를 독립항으로 청구하는 방안 검토

### 5.4 추가 조사 권장 사항

- Modbus 확장 주소 지정 관련 IETF/IEC 표준 문헌 추가 검토
- IEC 61850 (전력 시스템 통신 표준) 관련 선행기술 추가 검토
- 국내 HVAC 제조사(LG, 삼성, 캐리어 등) 보유 특허 추가 검토
- Electron 기반 산업용 HMI 관련 선행기술 추가 검토

# 요약서

**발명의 명칭:** 에이전트 기반 멀티프로토콜 통신 모니터링 시스템 및 방법

---

## 【요약】

에이전트 기반 프로토콜 추상화 계층을 이용하여 RS-485, UDP, TCP 이종 통신 프로토콜을 단일 인터페이스로 통합하고, 폴링 응답값 비교(diff)를 통해 최대 160개 산업용 제습기 장치의 상태 변화를 자동 탐지하여 이중 음향 경보 및 영속 이벤트 로그로 통합 모니터링하는 시스템 및 방법.

---

## 【대표도면】

도 1 (시스템 전체 구성도)

---

## 요약서 상세 설명

### 해결하려는 과제

산업 현장에서 운용되는 복수의 제습기 장치는 RS-485, UDP, TCP 등 이종(異種) 통신 프로토콜로 연결되어 있으며, 이벤트 발행(publish) 기능이 없는 레거시 RS-485 장치는 상태 변화를 능동적으로 알릴 수 없다. 기존 시스템은 프로토콜별 독립 관제 소프트웨어를 별도로 운영하거나, 전용 하드웨어 게이트웨이를 도입해야 하여 구축 비용이 높고 통합 운영이 어려웠다. 또한 분할 수신되는 RS-485 직렬 스트림 데이터의 신뢰성 있는 재조립과 장치 오류 발생 시 실시간 청각 경보 제공이 과제로 남아 있었다.

### 주요 기술적 특징

본 발명은 다음의 기술적 구성요소를 유기적으로 결합한다.

**1. 에이전트 기반 멀티프로토콜 추상화 계층**
RS-485Agent, UDPAgent, TCPAgent 각각이 `IProtocolAgent` 공통 인터페이스를 구현하며, 상위 모니터링 계층은 프로토콜 종류와 무관하게 동일한 방식으로 장치와 상호작용한다. Electron contextBridge를 통한 보안 격리(IPC 경계)로 하드웨어 접근과 UI 프로세스를 분리한다.

**2. 그룹-장치 1바이트 계층 주소 체계**
단일 바이트(8비트)의 상위 4비트를 그룹 ID(0~9), 하위 4비트를 장치 인덱스(0~15)로 분할하여 최대 160개 장치를 계층적으로 식별한다. `flatIndex = groupId × 16 + deviceIndex` 공식으로 장치를 선형 인덱스에 매핑한다.

**3. 폴링-diff 기반 상태 변화 이벤트 자동 탐지**
1500ms 주기의 폴링 타이머로 각 장치에 상태 요청 패킷(CMD=0x91)을 전송하고, 수신된 응답 데이터를 직전 폴링 결과와 필드 단위로 비교(diff)한다. 운전 시작/정지, 압축기 상태, 에러 코드 등의 변화가 감지되면 이벤트를 자동 생성하여 이벤트 발행 기능이 없는 레거시 장치에서도 사실상 이벤트 드리븐 감시를 실현한다.

**4. 스트림 누적 버퍼 파싱 및 CRC-16 CCITT 검증**
OS의 직렬 포트 수신 이벤트에 걸쳐 분할 도착하는 RS-485 패킷을 내부 수신 버퍼(rxBuffer)에 누적한 후, 응답 헤더 시그니처(CMD=0xC1, LEN=0x0D)를 탐색하여 18바이트 완전 프레임을 추출하고 CRC-16 검증을 통과한 경우에만 파싱을 수행한다. 불완전 데이터는 버퍼에 유지되어 후속 수신 데이터와 결합된다.

**5. 이중 음향 경보 아키텍처**
이벤트 발생 시 AudioContext API를 통해 800Hz 단발 비프음(이벤트 비프, 100ms)을 재생하며, 임계적 오류 상태(errorCode ≠ 0)가 지속되는 동안 600Hz 경고음(150ms)을 1초 주기로 반복 재생한다. 두 채널은 독립된 상태 머신으로 관리되어 동시 활성화가 가능하며 각각 사용자 설정으로 개별 제어된다.

**6. TSV 형식 이벤트 로그 영속 저장**
발생한 이벤트는 타임스탬프, 장치 위치, 장치 ID, 메시지 등을 탭 구분자(TSV) 형식으로 로컬 파일 시스템에 누적 저장하여 이력 조회 및 CSV 내보내기가 가능하다.

### 주요 효과

- 범용 PC의 단일 소프트웨어(Electron 애플리케이션)만으로 전용 게이트웨어 하드웨어 없이 멀티프로토콜 통합 모니터링 환경을 구축할 수 있다.
- 이벤트 발행 기능이 없는 레거시 RS-485 제습기 장치에 대해 폴링-diff 방식으로 이벤트 드리븐 수준의 실시간 상태 감시를 실현한다.
- 이중 음향 경보(800Hz 이벤트 비프 + 600Hz 연속 경고음)로 운영자가 현장을 떠나 있어도 장치 이상을 즉시 인지할 수 있다.
- 인터넷 연결 없이 완전 로컬 동작하며, Electron contextBridge 보안 격리로 하드웨어 접근 권한과 UI 렌더러를 분리하여 보안성을 확보한다.
